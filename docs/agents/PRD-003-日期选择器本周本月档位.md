# PRD: 日期选择器 — 本周 / 本月 / 全部档位修复

> 创建日期：2026-05-11
> 状态：`ready-for-agent`

---

## Problem Statement

用户打开首页日期选择器时，只能看到逐日档位（6 天前～今天）。**「本周」「本月」两个档位虽然已被 `getDateFilters()` 生成并渲染在 ScrollView 中，但由于毛玻璃渐变遮罩 `.idx-date-overlay` 在右侧 75%–100% 施加了实色暗部（`#09090b`），这两个档位在视觉上完全不可见。** 同时，「全部」档位在 i18n 已定义但从未加入选项列表。

用户期望：在「今天」的右侧能**清晰看到**「本周」「本月」两个快捷档位，并能点击使用。

---

## Solution

### 核心思路

不增加新 UI 组件，通过 **3 处 CSS + 1 处逻辑 + 1 处数据** 最小化改动，让被渐变遮罩掩盖的档位重新可见。

### 方案

1. **收窄渐变暗区**：`.idx-date-overlay` 右侧暗部起止从 `75%–100%` 调整为 `88%–100%`，释放右侧 ~13% 的可视空间
2. **增大尾部留白**：尾部 spacer 从 `halfScreen - 38` 增大到 `halfScreen - 12`，多出 26px 把「本月」推入高亮可视区
3. **补全「全部」档位**：`getDateFilters()` 末尾追加 `{ key: 'all', ... }`，并传递 i18n label
4. **同步 light theme 渐变**：`.idx-light` 下的 overlay 渐变同步收窄
5. **吸附边界兜底**：`doSnap` 已通过 `Math.min(nearestIndex, dateOptions.length - 1)` 保护边界，无需额外改动

---

## User Stories

1. As a 用户, I want **看到「本周」档位** in the date picker, so that I can filter news from the past 7 days with one tap
2. As a 用户, I want **看到「本月」档位** in the date picker, so that I can filter news from the current month with one tap
3. As a 用户, I want the new date options **positioned to the right of 「今天」**, consistent with the left-to-right chronological order
4. As a 用户, I want **frosted-glass visual effect preserved** for date options outside the center focus area, so that the overall aesthetic remains intact
5. As a 用户 in light mode, I want the **same visibility improvements** applied to the light theme gradient overlay
6. As a 用户, I want to **scroll the picker** and have snap behavior work correctly for the newly-visible week/month items
7. As a 用户, I want to **tap「本周」or「本月」** and immediately see filtered news results
8. As a 用户 in English locale, I want to see **"Week"** and **"Month"** labels when language is set to English
9. As a 用户, I want a **「全部」option** to remove all date filtering and see all news
10. As a 用户, I want the **initial load** to still center on 「今天」, with「本周」「本月」naturally visible to the right without needing to scroll

## Implementation Decisions

### Module 1: 渐变遮罩 — `idx-date-overlay` (index.scss)

- 深色主题 `.idx-date-overlay`: 右侧暗区 `#09090b 75%, #09090b 100%` → `#09090b 88%, #09090b 100%`
- 亮色主题 `.idx-light .idx-date-overlay`: 对应色值同步调整 `#ffffff 88%, #ffffff 100%`
- 左侧暗区保持不变（0%–25%），仅影响右侧
- 渐变过渡区自动拉伸至 40%–88%，保持平滑

### Module 2: 尾部 spacer 宽度 (index.tsx)

- 当前首尾 spacer 对称：`halfScreen - 38`
- 尾部 spacer 改为 `halfScreen - 12`，多出 26px 有效可视空间
- 确保「本月」（索引 8）在初始 scrollLeft 下能从高亮区看到，不落入右侧暗区
- 首部 spacer 不变，保持「今天」初始居中

### Module 3: 「全部」档位 (api/index.ts + index.tsx)

- `getDateFilters()` 末尾追加选项：
  ```
  { key: 'all', label: t.all || '全部', dayLabel: 'ALL', value: '∞', isSpecific: false }
  ```
- `index.tsx` 调用 `getDateFilters` 时补充 `all` 标签：
  ```
  getDateFilters({
    today: t('dateToday'),
    yesterday: t('dateYesterday'),
    week: t('dateWeek'),
    month: t('dateMonth'),
    all: t('dateAll'),
  })
  ```
- `isInDateRange(dateStr, 'all')` 已有 `if (filter === 'all') return true`，无需改动

### Module 4: 吸附边界（无需改动）

- `doSnap` 已通过 `Math.min(nearestIndex, dateOptions.length - 1)` 保证 index 不越界
- `handleDateScroll` 已通过相同逻辑做实时筛选更新
- 新增 all 档位后 `dateOptions.length` 自动增大，边界保护天然生效

```

```

- 仅测试外部可观测行为：档位是否可见、是否可点击、筛选是否正确

## Out of Scope

- 日期选择器改为下拉菜单或日历组件
- 调整档位顺序（保持逐日 → 昨天 → 今天 → 本周 → 本月 → 全部 的时序）
- 增加「自定义日期范围」功能
- 调整渐变中间过渡区颜色或曲线
- `isInDateRange` 逻辑变更（本周/本月/全部 的实现已正确）

---

## Further Notes

- 改动总量小（~15 行），三个文件，风险低
- 渐变暗区收窄到 88% 是权衡值：过宽（如 92%）会让「本月」仍然处于半透明边缘；过窄（如 98%）会让最右项过于突兀。88% 对应「本月」标签完整落入 50%–88% 高亮过渡区间
- 尾部 spacer 从 `-38` 改为 `-12` 相当于每个 item 右侧多暴露 26px —— 原本 `38` 是为了给 date-glow（65px 宽）留更多边缘空间，现改为 12px 仍然在 glow 宽度范围内
